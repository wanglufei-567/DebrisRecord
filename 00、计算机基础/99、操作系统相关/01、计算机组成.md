## 计算机系统

### 一、前言

**计算机系统从组成边界理解如下**👇：

```text
计算机系统 = 硬件 + 操作系统 + 应用程序
```

更准确的结构是：

```text
计算机系统
├── 硬件
│   ├── CPU
│   ├── 内存
│   ├── 磁盘
│   └── 网卡
│
├── 操作系统
│   ├── 内核
│   │   ├── 进程调度
│   │   ├── 内存管理
│   │   ├── 文件系统
│   │   ├── 网络协议栈
│   │   └── 设备驱动
│   │
│   └── OS 自带的用户空间组件
│       ├── 系统库
│       ├── Shell
│       ├── 系统服务
│       ├── 基础命令
│       └── GUI / 桌面环境
│
└── 普通应用程序
    ├── 浏览器
    ├── IDE
    ├── 微信
    ├── 游戏
    └── 业务服务
```

这里最容易混淆的是**「用户空间」**和**「操作系统」**的关系：

- **「用户空间」**是普通用户态程序运行的虚拟地址空间范围，它不是某一个软件包，也不等于操作系统

- **「操作系统」**只包含其中一部分用户空间组件

  - 例如 **Linux** 发行版里常见的：

    - `bash`

    - `ls`

    - `systemd`

    - `glibc`

    - `ssh`

    - 包管理器

    - 桌面环境


  - 这些组件虽然运行在**「用户空间」**，但通常属于操作系统发行版的一部分


  - 普通应用程序也运行在**「用户空间」**，但不属于操作系统

    - 例如：**Chrome**、**VS Code**、**Node.js 业务服务**


所以，狭义操作系统主要指内核；广义操作系统通常包括内核以及一组基础用户空间组件。普通应用程序也运行在用户空间，但不属于操作系统本身

------

**计算机系统从运行空间理解如下**👇：

```text
计算机系统 = 硬件 + 内核空间 + 用户空间
```

更准确的结构是：

```text
计算机系统
├── 硬件
│   ├── CPU
│   ├── 内存
│   ├── 磁盘
│   └── 网卡
│
├── 内核空间
│   └── 操作系统内核
│       ├── 管理硬件
│       ├── 管理进程
│       ├── 管理内存
│       ├── 管理文件
│       ├── 管理网络
│       └── 提供系统调用
│
└── 用户空间
    ├── OS 基础用户空间组件
    │   ├── 系统库
    │   ├── Shell
    │   ├── 系统服务
    │   ├── 基础命令
    │   └── GUI
    │
    └── 普通应用程序
        ├── 浏览器
        ├── IDE
        ├── 语言运行时
        ├── 业务服务
        └── 桌面应用
```

最关键的调用关系是：

```text
普通应用程序
    ↓
用户空间组件：运行时 / 系统库 / Shell / GUI
    ↓
系统调用
    ↓
内核空间：操作系统内核
    ↓
硬件
```

也可以记成：

```text
硬件：提供资源
内核：管理资源
用户空间：运行程序
系统调用：用户空间进入内核的入口
```

### 二、常见的操作系统

常见的桌面和服务器操作系统主要包括 **Windows**、**macOS**、**Linux**

#### 2.1、Windows

**Windows** 是 **Microsoft** 推出的操作系统，常见于个人电脑、办公电脑、游戏电脑和部分企业环境

它的特点是：

- 图形界面成熟，普通用户上手成本较低
- 商业软件、办公软件和游戏生态丰富
- 硬件厂商适配较多，驱动支持相对完善
- 系统整体由 **Microsoft** 统一设计、发布和维护

#### 2.2、macOS

**macOS** 是 **Apple** 推出的操作系统，只运行在 **Mac** 设备上

它的特点是：

- 硬件和系统由 **Apple** 统一设计，整体体验一致性较强
- 图形界面、字体渲染、触控板、生态联动体验较好
- 常用于设计、开发、音视频创作等场景
- 系统封闭程度高于普通 **Linux** 发行版，硬件选择范围也更受限制

#### 2.3、Linux

**Linux** 严格来说主要指 **Linux** 内核

日常所说的 **Linux** 系统，通常不是只有内核，而是由两部分组成：

```text
Linux 系统 = Linux 内核 + 发行版程序
```

其中：

- **Linux** 内核负责进程、内存、文件系统、网络、设备驱动等底层能力
- 发行版程序负责把内核、系统库、Shell、基础命令、包管理器、系统服务、桌面环境等组件组合起来，形成一个可安装、可使用的完整操作系统

常见的 **Linux** 发行版包括：

- **Ubuntu**：使用门槛较低，桌面、服务器和开发环境中都很常见
- **Debian**：社区历史较长，强调稳定性，也是很多发行版的基础
- **Fedora**：更新节奏较快，常用于体验较新的 **Linux** 技术
- **Red Hat Enterprise Linux**：面向企业服务器和生产环境
- **openSUSE**：常用于桌面和服务器场景，工具链较完整
- **Arch Linux**：滚动更新，适合希望深入理解系统配置和组件关系的用户

所以，**Windows** 和 **macOS** 更像是由厂商统一发布的完整操作系统；而日常所说的 **Linux** 系统，通常是基于 **Linux** 内核构建出来的不同发行版

## 大文件上传

大文件上传最成熟的解决方案是：**文件切片（Chunking） + 并发控制 + 断点续传（Breakpoint Resumption）**。

核心原理：将大文件拆分为多个小块（**Chunks**），利用 `Blob.prototype.slice()` 方法进行切割

1. **文件切片**：通过 `Blob.slice` 将大文件拆分成固定大小（例如 2MB）的 `Blob` 对象
2. **生成 Hash**：使用 `SparkMD5` 计算文件的唯一标识，用于服务端区分文件及实现秒传
3. **分片上传**：通过 `FormData` 将每个分片上传至服务器
4. **合并请求**：所有分片上传完成后，通知服务端合并分片

完整代码实现👇：

```JavaScript
// 假设这是上传组件的核心逻辑
const CHUNK_SIZE = 2 * 1024 * 1024; // 2MB 每个分片
const UPLOAD_URL = 'https://mock-server.com/upload';
const MERGE_URL = 'https://mock-server.com/merge';

/**
 * 1. 文件切片
 */
function createChunks(file) {
  // 切片数组
  const chunks = [];
  let cur = 0;
  while (cur < file.size) {
    chunks.push(file.slice(cur, cur + CHUNK_SIZE));
    cur += CHUNK_SIZE;
  }
  return chunks;
}

/**
 * 2. 并发上传控制 (核心面试加分项)
 * limit: 最大并发数
 */
async function uploadChunks(chunks, fileHash, limit = 3) {
  const tasks = [];
  const results = [];
  
  for (let i = 0; i < chunks.length; i++) {
    const chunk = chunks[i];
    const formData = new FormData();
    formData.append('file', chunk);
    formData.append('hash', fileHash);
    formData.append('index', i);

    // 创建上传任务
    const task = fetch(UPLOAD_URL, {
      method: 'POST',
      body: formData,
    }).then(() => {
      console.log(`分片 ${i} 上传成功`);
      results.push(i);
    });

    tasks.push(task);

    // 当并发数达到 limit 时，等待其中一个完成再继续
    if (tasks.length >= limit) {
      await Promise.race(tasks);
      // 这里简写，实际生产中需配合数组过滤已完成的任务
    }
  }

  await Promise.all(tasks);
  
  // 3. 通知服务端合并
  await fetch(MERGE_URL, {
    method: 'POST',
    body: JSON.stringify({ fileHash, filename: 'example.mp4' })
  });
  console.log('所有分片已上传，请求合并完成');
}

// 调用示例

const fileInput = document.querySelector('input[type="file"]');
fileInput.onchange = (e) => {
  // 获取到 input 标签中上传的文件
  const file = e.target.files[0];
  const chunks = createChunks(file);
  // fileHash 通常使用 spark-md5 计算，此处简化处理
  uploadChunks(chunks, 'unique-file-hash-123');
};
```

## 开发常用模块

### 一、第三方依赖管理

**相关工具**

- `venv`：**Python** 标准库里的虚拟环境工具，用来隔离当前项目的第三方依赖，避免污染全局解释器环境
- `pip`：**Python** 包安装工具，负责把第三方包安装到当前解释器环境中
- `requirements.txt`：**pip** 常用依赖清单，可以手写，也可以由当前环境导出
- `pyproject.toml`：现代 **Python** 项目的标准项目配置文件，可以记录项目元数据、构建系统和依赖声明

**常用命令**

```bash
# 创建虚拟环境
python -m venv .venv

# 激活虚拟环境
source .venv/bin/activate

# 安装单个依赖
python -m pip install httpx

# 根据依赖清单安装依赖
python -m pip install -r requirements.txt

# 把当前环境中的精确版本导出为依赖清单
python -m pip freeze > requirements.txt
```

**依赖版本记录**

依赖清单 `requirements.txt`：适合简单脚本、传统 **pip** 项目，或者作为部署安装清单

```txt
# requirements.txt
httpx==0.27.2
rich>=13.7,<14
```

项目配置文件 `pyproject.toml` ：适合记录项目级依赖声明，是项目配置入口

```toml
# pyproject.toml
[project]
dependencies = [
  "httpx==0.27.2",
  "rich>=13.7,<14",
]
```

### 二、路径、文件与 JSON

**相关模块**

- `pathlib`：用面向对象方式处理文件路径，适合拼接路径、判断文件是否存在、读取和写入文本文件
- `json`：负责 **JSON** 编码和解码，适合在字符串、文件内容和 **Python** 数据结构之间转换

**常用 API**

- `Path(path)`：创建路径对象
- `path.exists()`：判断路径是否存在
- `path.read_text(encoding="utf-8")`：读取文本
- `path.write_text(text, encoding="utf-8")`：写入文本
- `json.loads(text)`：把 JSON 字符串解析成对象
- `json.dumps(data, ensure_ascii=False, indent=2)`：把对象序列化成字符串

**示例代码**

```python
# Python
import json
from pathlib import Path


def load_session(path: str) -> dict:
    text = Path(path).read_text(encoding="utf-8")
    return json.loads(text)
```

```js
// JavaScript
import { readFile } from "node:fs/promises";

async function loadSession(path) {
  const text = await readFile(path, "utf-8");
  return JSON.parse(text);
}
```

### 三、解释器运行环境

**相关模块**

- `sys`：内置标准库模块，当前 **Python** 解释器进程的运行上下文接口

  - 接口提供：命令行参数、解释器路径、模块搜索路径、已加载模块缓存、标准输入输出和进程退出方式

    > **作用机制**： **Python** 解释器进程启动后，**Python Runtime** 会先完成一批初始化工作，再执行目标脚本或目标模块
    >
    > - 解析启动命令，把脚本名和参数放进 `sys.argv`
    > - 确定当前解释器位置，放进 `sys.executable`
    > - 计算模块搜索路径，放进 `sys.path`
    > - 准备标准输入、标准输出和错误输出，放进 `sys.stdin`、`sys.stdout`、`sys.stderr`
    > - 建立模块缓存表，放进 `sys.modules`
    >
    > 所以 `sys` 是观察和影响当前解释器进程的入口
    >

**常用 API**

- `sys.path`：模块搜索路径列表，代码中的 `import` 会基于这里的目录查找模块
  - `sys.path.insert(0, path)`：把某个目录放到模块搜索路径最前面，提高本地模块解析优先级

- `sys.modules`：当前进程已经加载过的模块缓存，重复 `import` 通常会先从这里取
- `sys.argv`：当前脚本收到的命令行参数
- `sys.executable`：当前正在运行的 **Python** 解释器路径
- `sys.version`：当前 **Python** 版本信息
- `sys.platform`：当前运行平台，例如 `darwin`、`linux`、`win32`
- `sys.exit(code)`：主动结束当前进程
- `sys.stdin`、`sys.stdout`、`sys.stderr`：标准输入、标准输出、错误输出

**示例代码**

```python
# Python
import importlib
import sys
from pathlib import Path

# __file__ 表示当前代码文件路径 .parent 取它所在的目录
ROOT = Path(__file__).resolve().parent

# sys.path 决定 import 的搜索目录
# 若是当前目录不在 sys.path 中，则将其插入到 sys.path 最前面
if str(ROOT) not in sys.path:
    sys.path.insert(0, str(ROOT))

# 以当前目录为前缀，动态 import 目标模块
module = importlib.import_module("plugins.tools")

# sys.argv 查看启动参数
print("argv:", sys.argv)

# sys.executable 查看当前解释器
print(sys.executable)
print("module:", module.__name__)

# sys.modules 查看已加载模块
print("loaded:", "plugins.tools" in sys.modules)
```

```js
// JavaScript
// Node.js 中类似能力主要分散在 process 和模块系统里
console.log(process.argv);
console.log(process.execPath);
console.log(process.platform);
```

### 四、环境变量与进程信息

**相关模块**

- `os`：提供操作系统相关接口，常用于读取环境变量、当前工作目录和进程运行环境信息，接近 **Node.js** 的 `process`

**常用 API**

- `os.environ["KEY"]`：读取环境变量，不存在会抛错
- `os.environ.get("KEY")`：安全读取环境变量
- `os.getcwd()`：读取当前工作目录
- `os.chdir(path)`：切换当前工作目录

**示例代码**

```python
# Python
import os

api_key = os.environ.get("OPENAI_API_KEY")
cwd = os.getcwd()
```

```js
// JavaScript
const apiKey = process.env.OPENAI_API_KEY;
const cwd = process.cwd();
```

### 五、子进程执行

**相关模块**

- `subprocess`：用于从当前 **Python 进程**启动外部命令，并读取命令退出码、标准输出和错误输出

**参数说明**

`subprocess.run(args, capture_output=False, text=False, timeout=None, check=False)` 常见参数如下

- `args`：命令和参数列表，推荐用 list
- `capture_output`：是否捕获 stdout / stderr
- `text`：是否按字符串处理输出
- `timeout`：超时时间，单位是秒
- `check`：命令非 0 退出时是否抛异常

**示例代码**

```python
# Python
import subprocess


completed = subprocess.run(
    ["python", "--version"],
    capture_output=True,
    text=True,
    timeout=5,
    check=True,
)

print(completed.stdout or completed.stderr)
```

```js
// JavaScript
import { execFile } from "node:child_process";
import { promisify } from "node:util";

const execFileAsync = promisify(execFile);

const result = await execFileAsync("node", ["--version"], {
  timeout: 5000,
});

console.log(result.stdout);
```

### 六、信号、线程与并发执行

**相关模块**

这些模块处理长期运行服务和阻塞任务

- `signal`：处理 `SIGTERM`、`SIGHUP`、`SIGINT`
- `threading`：提供线程基础能力，适合把阻塞任务放到后台线程执行
- `concurrent.futures`：更高层的线程池 / 进程池 API

这组模块在 **Agent** 中常用于阻塞工具隔离、后台任务和服务优雅关闭

**参数说明**

`ThreadPoolExecutor(max_workers=None)` 的核心参数如下

- `max_workers`：最多并发 worker 数量，过大可能造成资源争用

**示例代码**

```python
# Python
from concurrent.futures import ThreadPoolExecutor


def run_blocking_tool(name: str) -> str:
    return f"{name}:ok"


with ThreadPoolExecutor(max_workers=4) as executor:
    future = executor.submit(run_blocking_tool, "read_file")
    result = future.result(timeout=5)
```

```js
// JavaScript
// JavaScript 中类似问题通常会考虑 worker_threads、child_process 或任务队列
// 具体 API 和 Python 线程池并不一一对应
```

### 七、SQLite 与本地状态

**相关模块**

- `sqlite3`：**Python** 标准库里的本地嵌入式数据库能力，适合 **session store**、**memory store**、**job store**

**常用 API**

- `sqlite3.connect(path)`：打开数据库连接
- `conn.execute(sql, params)`：执行 **SQL**
- `conn.commit()`：提交事务
- `conn.close()`：关闭连接
- `with sqlite3.connect(path) as conn:`：用上下文管理连接

**示例代码**

```python
# Python
import sqlite3


with sqlite3.connect("agent.db") as conn:
    conn.execute(
        "create table if not exists sessions (id text primary key, content text)"
    )
    conn.execute(
        "insert or replace into sessions (id, content) values (?, ?)",
        ("s1", "{}"),
    )
```

```js
// JavaScript
// Node.js 通常使用 better-sqlite3、sqlite、drizzle 等第三方库
// 标准库没有内置 sqlite3 模块
```

### 八、CLI 与终端输出生态

**相关模块**

- `argparse`：**Python** 标准库里的 **CLI** 参数解析工具，适合简单命令行脚本
- **Typer**：第三方 **CLI** 框架，适合基于类型标注快速构建命令行应用
- **Click**：第三方 **CLI** 框架，适合构建更复杂的命令、子命令和参数组合
- **Rich**：第三方终端输出库，适合表格、颜色、进度条和更可读的命令行输出

**示例代码**

```python
# Python
import argparse

parser = argparse.ArgumentParser()
parser.add_argument("--profile", default="default")
args = parser.parse_args()

print(args.profile)
```

```js
// JavaScript
const profileIndex = process.argv.indexOf("--profile");
const profile = profileIndex >= 0 ? process.argv[profileIndex + 1] : "default";

console.log(profile);
```

### 九、HTTP 与 Web 服务生态

**相关模块**

- `httpx`：第三方 **HTTP client**，用于请求外部接口，支持同步和异步 API
- **FastAPI**：第三方 **Web API** 框架，常用于构建 HTTP API、Webhook 和 provider gateway
- **Starlette**：更底层的 ASGI 框架，**FastAPI** 建立在它之上

在 provider wrapper 中，timeout 和错误处理是调用外部服务时最重要的工程边界

**httpx 常用参数**

- `url`：请求地址
- `headers`：请求头
- `json`：JSON 请求体
- `timeout`：超时时间

**示例代码**

```python
# Python
import httpx


def call_provider(api_key: str, prompt: str) -> dict:
    response = httpx.post(
        "https://api.example.com/chat",
        headers={"Authorization": f"Bearer {api_key}"},
        json={"prompt": prompt},
        timeout=30,
    )
    response.raise_for_status()
    return response.json()
```

```js
// JavaScript
async function callProvider(apiKey, prompt) {
  const response = await fetch("https://api.example.com/chat", {
    method: "POST",
    headers: {
      Authorization: `Bearer ${apiKey}`,
      "Content-Type": "application/json",
    },
    body: JSON.stringify({ prompt }),
  });

  if (!response.ok) {
    throw new Error(`provider failed: ${response.status}`);
  }

  return response.json();
}
```

## 虚拟环境、依赖管理与代码质量工具

### 一、虚拟环境

#### 1.1、解决什么问题

虚拟环境用于给每个 **Python** 项目准备==一套独立的解释器环境和依赖目录==

如果不使用虚拟环境，多个项目可能共享同一批第三方包，容易出现版本冲突

例如：

- 项目 A 需要 `requests==2.31.0`
- 项目 B 需要 `requests==2.32.0`
- 如果都安装到全局环境，后安装的版本可能影响另一个项目

虚拟环境的核心作用是：把项目依赖限制在当前项目目录里

#### 1.2、常用命令

```zsh
# 创建虚拟环境
# -m venv：以模块方式运行标准库 venv，用它创建虚拟环境
# .venv：虚拟环境目录名，前面的 . 表示隐藏目录
python -m venv .venv

# macOS / Linux 激活虚拟环境：
# .venv/bin/activate：虚拟环境里的激活脚本，激活后，python 和 pip 会优先指向 .venv 里的版本
source .venv/bin/activate

# 查看当前 shell 实际使用的 Python 路径，用来确认是否已切到 .venv
which python

# 退出虚拟环境，恢复到激活前的 shell 环境
deactivate
```

激活虚拟环境后，再执行 `python`、`pip`，通常会优先使用 `.venv` 里的版本

#### 1.3、目录结构

```text
project/
  .venv/
  app/
  tests/
  requirements.txt
```

`.venv/` 通常不提交到 **Git**，因为它是本机生成的环境目录

### 二、依赖管理

#### 2.1、解决什么问题

依赖管理用于记录项目需要哪些第三方包，以及这些包应该如何安装

常见做法有两类

- 简单项目：使用 `requirements.txt`（依赖清单）
- 工程化项目：使用 `pyproject.toml`（项目级工程配置）

#### 2.2、使用 requirements.txt

安装依赖

```zsh
pip install requests
```

查看当前环境中安装的包

```zsh
pip list
```

把当前依赖写入 `requirements.txt`

```zsh
pip freeze > requirements.txt
```

从 `requirements.txt` 安装依赖

```zsh
pip install -r requirements.txt
```

`requirements.txt` 示例

```text
requests==2.32.3
pytest==8.2.2
```

`==` 表示锁定具体版本，这样其他人安装时更容易得到一致的环境

#### 2.3、pyproject.toml

`pyproject.toml` 是 **Python** 项目里更现代的工程配置入口，常用于记录项目元信息、依赖、构建配置、格式化配置和检查规则

示例：

```toml
[project]
name = "example-app"
version = "0.1.0"
dependencies = [
  "requests>=2.32.0",
]
```

### 三、格式化与静态检查

#### 3.1、工具分类

格式化、静态检查、类型检查都属于代码运行前的质量检查

它们不负责执行业务逻辑，主要在提交代码、运行测试、发布部署之前发现工程质量问题

- 格式化：统一代码排版，例如缩进、换行、空格、行宽
- 静态检查：扫描源码中的明显问题，例如未使用变量、导入顺序、错误写法
- 类型检查：读取类型标注，检查参数、返回值、对象字段、模块边界是否匹配

三类工具的分工如下

| 类型 | 解决的问题 | 常见工具 |
| --- | --- | --- |
| 格式化 | 代码长什么样 | `ruff format` |
| 静态检查 | 代码有没有明显问题 | `ruff check` |
| 类型检查 | 类型标注和调用关系是否一致 | `mypy`、`pyright` |

#### 3.2、工具来源

这些工具通常不是 **Python** 解释器内置能力，而是项目额外安装的开发依赖

- `ruff`：第三方工具，可以同时做格式化和静态检查
- `mypy`：第三方类型检查工具，常用于命令行和 **CI**
- `pyright`：第三方类型检查工具，常和编辑器集成，用于实时提示

```zsh
# 安装 ruff 和 mypy
pip install ruff mypy
```

`pyproject.toml` 中可以集中放工具配置

```toml
[tool.ruff]
line-length = 100

[tool.mypy]
python_version = "3.12"
strict = true
```

#### 3.3、手动执行

```zsh
# 格式化当前项目
ruff format .

# 静态检查当前项目
ruff check .

# 类型检查当前项目
mypy .
```

命令里的 `.` 表示当前目录，也就是当前项目

检查指定目录时，把 `.` 换成具体路径

```zsh
# 只检查 app 目录
ruff check app
mypy app
```

#### 3.4、自动触发

自动触发指工具由编辑器、**Git** 钩子或 **CI** 流程执行

- 编辑器保存时自动执行 `ruff format`
- **Git** commit 前通过 pre-commit 执行 `ruff check`
- **CI** 中执行 `ruff check .` 和 `mypy .`
- 团队项目通过 `pyproject.toml` 统一规则

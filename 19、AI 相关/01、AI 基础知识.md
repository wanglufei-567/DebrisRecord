## AI 基础知识

### 一、基础概念

- **模型（Model）**

  - **定义**：**==模型是一个数学函数或算法，用于将输入映射到输出==**

    - **简单函数**：在数学中，函数 $f$ 的作用是输入一个值 $x$，经过某种规则，得到输出 $y$
      - 比如 $y = 2x$，如果输入 $2$，输出一定是 $4$，这个规则是人为指定的
    - **AI 模型函数：** ==AI 模型的本质是一个极其复杂的函数==，这个函数的“**内部规则**”不是人写死的，而是由数亿个可调节的数字（参数）组成的
      - ==训练的过程，就是不断调整这些参数，直到函数能准确地把输入映射到输出==
      - 比如图像识别模型，输入 $x$ 是成千上万个像素点的数值，输出 $y$ 是“猫”或“狗”的标签

  - **本质**：==一个带参数的函数，通过大量数据的训练，不断调整参数，使它的输出尽量接近真实结果==

    - AI 不是直接给出一个确定答案，而是为可能的结果分配概率，并根据这个概率分布生成输出（<!--有时会选择概率最高的结果，有时会按概率进行随机选择-->）因此本质上是在处理**「概率分布」**

    -  比如语言模型，输入“今天天气很”，模型并不是“思考”后回答，而是在计算下一个字出现的概率：
      - “好”的概率：$80\%$
      - “差”的概率：$15\%$
      - “吃”的概率：$5\%$
    -  概率越高，被选中的可能性越大，但最终输出通常是“按概率随机抽样”得到的，而不是固定选择最大值

-  **大模型（Large Model / LLM）**

  - 大模型和普通模型的主要区别就是参数量更大
    - 传统模型可能只有几百万个参数，而 LLM 的参数通常在百亿（10B）到万亿级
  - 大模型的训练数据规模更大，覆盖更丰富的语言场景

- **参数（Parameters）**

  - **定义**：模型内部可调节的**权重**，决定模型如何将输入映射为输出
    - 如果把模型比作一个巨大的**收音机**，参数就是面板上数以亿计的**旋钮**
  - **作用**： 训练模型的过程，就是不断旋转这些旋钮，直到模型能够准确地预测下一个字
    - 参数越多，模型容量越大，理论上能“记住”更多语言规律和知识

- **Token 与 Tokenization**

  - **Token**：文本的最小处理单元，它可以是一个单词、一个汉字，甚至是一个字符碎片
  - **Tokenization**：将输入文本拆分为模型能理解的 **Token**

- **Transformer** 

  - 一种处理序列数据（尤其是文本）的神经网络架构

  - 它可以**理解和生成序列数据（尤其是自然语言）** <!--理解和生成语言-->

    具体可以做：<!-- 几乎所有“和语言相关的 AI 能力”，Transformer 都可以支撑-->

    - 文本生成（对话、写作、代码生成）
    - 文本理解（分类、情感分析、问答）
    - 翻译（多语言互译）
    - 摘要（长文压缩）
    - 多模态（文本 + 图像 + 音频）

### 二、LLM 工程化中的概念

> ==**技术发展的中间产物**==

- **Prompt & Context & Memory**

  - **Prompt**：给模型的“指令 + 输入”

  - **Context**：**当前这轮对话里模型能看到的全部信息（短期记忆）**
    - 不仅包括当前的 Prompt，还包括：
      - 之前说过的话（对话历史）
      - 系统设定（比如角色设定）
      - 已生成的内容
  - **Memory**：**跨对话、长期保存的信息（长期记忆）**
    - 为什么需要 **Memory**？
      - 上下文长度有限，Context 不能无限长（有 token 限制），长对话会被截断
      - 每次新对话，模型默认不了解你是谁，不会记住你的偏好、历史信息
    - **Memory** 工作流程
      1. 从对话中提取重要信息
      2. 存储（数据库 / 向量库等）
      3. 下次对话时检索相关内容
      4. 拼接进 **Context**
      5. 模型基于“补全后的 **Context**”回答
    - ==工程化中 **Memory** 需要解决的三个具体问题：==
      1. 存什么（信息抽取）
      2. 怎么存（结构化 vs 向量化）
      3. 怎么取（检索策略）

- **RAG（Retrieval-Augmented Generation）检索增强生成**

  - 为了解决 LLM 的“信息封闭”问题：
    - 模型不知道最新信息
    - 模型不知道私有数据

  - **RAG** 就是把外部数据临时“喂给模型”，再生成答案
    1. 从外部数据库中检索相关内容
    2. 把检索到的内容，加入到 **Context** 里
  - 和 **Memory** 的关系
    - **Memory**：记“用户相关信息”（长期）
    - **RAG**：查“外部知识”（实时）

- **Agent** ：**能“自主决策 + 调用工具 + 多步执行”的 LLM 系统** <!--工具是帮助 LLM 与外部交互的-->

  - 区别于普通 **LLM**：

    - 普通 **LLM**：问一句 → 回一句
    - **Agent**：可以自己**拆任务**、**规划步骤**、**调用工具**、**循环执行**

  - 举例说明：一个任务：“帮忙订一张明天去北京的机票”

    - **Agent** 的行为可能是：
      - 理解需求（时间、地点）
      - 调用航班查询 API
      - 比较价格
      - 选择最优方案
      - 返回结果或执行预订

    - 核心能力：

      - **规划（Plan）**

      - **行动（Act）**

      - **反思（Reflect，可选）**

  - **本质**：**Agent = LLM + 工具 + 控制流程（循环）**

- **Function Calling**：**让 LLM “按格式调用函数/API” 的能力**

- **Skill：** **把“如何完成一类任务”封装成一个可复用能力单元**

  - 通常是把以下东西打包在一起：
    - 一段 **Prompt**（怎么问模型）
    - 一组工具（可调用的 **API** / **Function**）
    - 一套流程（怎么做这件事）
  - 组合起来，形成一个“能力模块”

- **MCP（Model Context Protocol**）：**让 LLM “统一访问外部工具和数据” 的标准协议**

  - **MCP** 解决的问题：
    - 每个工具接入方式都不一样
    - 每个系统都要单独适配
  -  **MCP** 的目标：**统一“模型如何访问外部能力”**

### 三、LLM 的短板

**LLM** 虽凭借海量训练数据拥有渊博知识，但存在两大核心局限：

- **信息封闭**：知识固化于训练截止时间，无法获取 **实时信息** 或 **私有数据**
- **行动缺失**：缺乏与外部系统的交互能力，无法借助工具完成特定任务

因此，这一阶段的 **LLM 应用 **多以 **对话机器人** 形式存在，难以真正融入人类真实任务场景

为了解决这两个核心局限，对应的解决方案应运而生：

- **信息封闭** → 解决 “**知识补给**” 的 **RAG**（检索增强）
- **行动缺失** → 实现 “**工具交互**” 的 **Agent** / **Tool Calling**

### 四、Agent

#### 4.1、Agent 的本质

**Agent 是一个能 “自主决策 + 调用工具 + 多步执行” 的 LLM 系统**

**Agent** 的工作流程：**理解 → 规划 → 行动 → 观察 → 再决策（循环）**

**Agent** 本质上就是依靠大模型的 **语义识别** 和 **语言生成** 能力进行 **工程化开发**

### 五、多模态

**多模态（`Multimodality`）**指的是 **AI** 系统能够同时理解、处理和融合来自多种不同类型数据（文本、图像、视频、音频）的信息，并基于这些综合信息进行思考、推理和生成

**AI** 与人类的交互方式非常“机械”，必须用它能理解的方式（通常是文本或特定格式的图像）去“喂”给它信息，而人类认知世界的核心方式恰恰是多模态的

我们看一部电影，会同时处理画面（视觉）、对白和配乐（听觉），甚至结合字幕（文本），最终形成一个完整、连贯的理解

**AI 多模态**的目标，就是让机器尽可能地复现这种综合感知能

## Python 数据结构、类型标注与模块系统

### 一、List 列表

#### 1.1、语法与用法

`list` 是有序、可变集合，接近 **JavaScript** 的 `Array`

- `items.append(value)`：追加一个元素
- `items.extend(values)`：追加多个元素
- `len(items)`：获取长度
- `items[index]`：按索引读取
- `items[start:end]`：切片，左闭右开

#### 1.2、示例代码

```python
# Python
messages: list[dict[str, str]] = []

messages.append({"role": "user", "content": "hello"})
messages.extend([{"role": "assistant", "content": "hi"}])

latest_message = messages[-1]
recent_messages = messages[-2:]
```

```js
// JavaScript
const messages = [];

messages.push({ role: "user", content: "hello" });
messages.push({ role: "assistant", content: "hi" });

const latestMessage = messages.at(-1);
const recentMessages = messages.slice(-2);
```

### 二、Dictionary 字典

#### 2.1、语法与用法

`dict` 是键值映射，接近 **JavaScript** 的普通对象或 `Map`，常用于配置、工具参数、JSON-like 数据

- `data[key]`：读取 key，不存在会抛 `KeyError`
- `data.get(key, default)`：安全读取
- `data.keys()`：读取所有 key
- `data.values()`：读取所有 value
- `data.items()`：遍历 key-value 对

#### 2.2、参数说明

`dict.get(key, default=None)` 的参数含义如下

- `key`：要读取的键
- `default`：键不存在时返回的默认值

#### 2.3、示例代码

```python
# Python
config: dict[str, object] = {
    "model": "gpt-example",
    "max_steps": 8,
}

model = config.get("model", "default-model")
max_steps = config["max_steps"]

for key, value in config.items():
    print(key, value)
```

```js
// JavaScript
const config = {
  model: "gpt-example",
  maxSteps: 8,
};

const model = config.model ?? "default-model";
const maxSteps = config.maxSteps;

for (const [key, value] of Object.entries(config)) {
  console.log(key, value);
}
```

### 三、Tuple 元组与 Set 集合

#### 3.1、语法与用法

- `tuple` 是有序、不可变集合，常用于固定结构返回值
- `set` 是无序、不重复集合，常用于去重和成员判断
- `value in collection` 用于判断成员是否存在

#### 3.2、示例代码

```python
# Python
result: tuple[str, int] = ("ok", 200)
# 解构赋值
status, code = result

enabled_tools: set[str] = {"read_file", "write_file"}

if "read_file" in enabled_tools:
    print("can read")
```

```js
// JavaScript
const result = ["ok", 200];
const [status, code] = result;

const enabledTools = new Set(["read_file", "write_file"]);

if (enabledTools.has("read_file")) {
  console.log("can read");
}
```

### 四、解包与切片

#### 4.1、语法与用法

- `a, b = pair`：按位置解包
- `first, *rest = items`：第一个值单独取出，其余放入列表 <!--剩余参数 rest-->
- `items[start:end]`：切片，包含 `start`，不包含 `end`
- `items[-1]`：读取最后一个元素

#### 4.2、示例代码

```python
# Python
messages = ["system", "user", "assistant"]

first, *rest = messages
last = messages[-1]
middle = messages[1:2]
```

```js
// JavaScript
const messages = ["system", "user", "assistant"];

const [first, ...rest] = messages;
const last = messages.at(-1);
const middle = messages.slice(1, 2);
```

### 五、类型标注

#### 5.1、语法与用法

**Python** 类型标注主要服务可读性和静态检查，默认不做运行时强制校验，接近 **TypeScript** 编译期类型约束

- `name: str`：变量或参数类型
- `def fn() -> str`：返回值类型
- `str | None`：值可以是字符串或 `None`
- `Any`：放弃具体类型约束
- `list[Message]`：列表元素类型
- `dict[str, Any]`：key 是字符串，value 任意

#### 5.2、示例代码

```python
# Python
from typing import Any

Message = dict[str, Any]

def build_user_message(content: str, metadata: dict[str, Any] | None = None) -> Message:
    return {
        "role": "user",
        "content": content,
        "metadata": metadata or {},
    }
```

```ts
// TypeScript
type Message = Record<string, unknown>;

function buildUserMessage(
  content: string,
  metadata: Record<string, unknown> | null = null,
): Message {
  return {
    role: "user",
    content,
    metadata: metadata ?? {},
  };
}
```

### 六、模块系统、包路径与项目布局

#### 6.1、导入语法

- `import module`：导入整个模块
- `from package import name`：从包中导入指定名称
- `from .module import name`：相对导入，常见于包内部
- `__init__.py`：标识包边界，也可暴露包级 API
- 循环导入是大型项目常见风险

#### 6.2、标准库导入示例

```python
# Python
import json
from pathlib import Path

def load_json(path: str) -> dict:
    text = Path(path).read_text(encoding="utf-8")
    return json.loads(text)
```

```js
// JavaScript
import { readFile } from "node:fs/promises";

async function loadJson(path) {
  const text = await readFile(path, "utf-8");
  return JSON.parse(text);
}
```

#### 6.3、包路径与相对导入

**Python** 的导入不是单纯按文件路径拼接，而是按模块搜索路径和包结构解析

真实项目里通常优先使用包路径组织代码，让入口模块、业务模块和工具模块之间保持稳定导入关系

- `app/`：应用包目录
- `__init__.py`：标识包边界，也可以暴露包级 API
- `from app.tools.registry import load_tools`：绝对导入，适合跨目录引用
- `from .schema import ToolCall`：相对导入，适合同一个包内部模块互相引用
- `python -m app.main`：按模块路径启动入口，更适合保留包导入上下文

#### 6.4、项目布局示例

```text
project/
  app/
    __init__.py
    main.py
    tools/
      __init__.py
      registry.py
      schema.py
```

`app/main.py` 可以使用绝对导入

```python
# Python
from app.tools.registry import load_tools


def main() -> None:
    tools = load_tools()
    print(tools)
```

`app/tools/registry.py` 可以在包内部使用相对导入

```python
# Python
from .schema import ToolCall


def load_tools() -> list[ToolCall]:
    return [{"name": "read_file", "enabled": True}]
```

启动时优先使用模块路径

```zsh
python -m app.main
```

这样入口模块仍然处在 `app` 包上下文里，相对导入和绝对导入都更稳定

### 七、TypedDict 与结构化字典

#### 7.1、语法与用法

`TypedDict` 是 **Python** 标准库 `typing` 模块里的类型工具，不是 `list`、`dict`、`str` 这种无需导入的内置类型

使用前需要显式导入

```python
from typing import NotRequired, TypedDict
```

`TypedDict` 类似于 **TypeScript** 中的 **interface**：

- `class Message(TypedDict):`：声明一个结构化字典类型 <!--定义一个 interface-->
- `role: str`：字段 `role` 必须是字符串
- `content: str`：字段 `content` 必须是字符串
- `NotRequired[T]`：表示字段可以不存在，常用于可选字段

#### 7.2、示例代码

```python
# Python
from typing import NotRequired, TypedDict


class Message(TypedDict):
    role: str
    content: str
    metadata: NotRequired[dict[str, object]]


def build_message(content: str) -> Message:
    return {
        "role": "user",
        "content": content,
    }
```

```ts
// TypeScript
type Message = {
  role: string;
  content: string;
  metadata?: Record<string, unknown>;
};

function buildMessage(content: string): Message {
  return {
    role: "user",
    content,
  };
}
```

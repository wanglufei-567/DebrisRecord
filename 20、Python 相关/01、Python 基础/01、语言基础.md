## Python 语言基础

### 一、解释器与 REPL

#### 1.1、解释器

**Python** 是一门编程语言，`.py` 文件只是保存 **Python** 代码的文本文件

计算机不能直接理解 `.py` 文件里的语法，所以需要一个程序来读取、解析并执行这些代码，这个程序就是 **Python 解释器**

`python` 命令，本质上就是在命令行里启动 **Python 解释器**

```zsh
python main.py
```

这条命令可以拆成两部分

- `python`：启动 **Python 解释器**
- `main.py`：把 `main.py` 这个文件交给解释器执行

> 可以类比 **Node.js**：`.js` 文件本身不会自己运行，需要 `node` 启动运行环境；`node` 内部使用 **V8** 执行 **JavaScript**
>
> 同理，`.py` 文件本身不会自己运行，需要 `python` 启动 **Python 解释器** 来读取和执行代码
>
> 通常在执行脚本时会加上 `-u` 参数：`python -u main.py`，这是因为：
>
> - `python` ：输出可能先进入缓冲区，稍后再真正显示
> - `python -u` ：输出尽量立即写出，不等缓冲区

#### 1.2、什么是 REPL

**REPL** 是 **Read**、**Evaluate**、**Print**、**Loop** 的缩写，表示一种交互式运行环境

在命令行直接输入 `python`，不带文件名，就会进入 **REPL**

进入 **REPL** 后，可以一行一行输入 **Python** 代码，解释器会立刻执行并显示结果

**REPL** 适合做这些事

- 快速验证一个表达式
- 测试一个函数调用
- 查看一个对象的类型和值
- 临时验证某段语法，不需要新建 `.py` 文件

它不适合承载完整项目逻辑，因为 **REPL** 里的代码不方便长期维护、复用和版本管理

#### 1.3、Python 三种常见执行方式

| 命令 | 含义 | 常见场景 |
| --- | --- | --- |
| `python main.py` | 按文件路径执行 `main.py` | 本地脚本、单文件练习 |
| `python -m package.module` | 按模块路径执行 `package.module` | 包内 **CLI** 入口、需要保持包导入关系的代码 |
| `python` | 进入 **REPL** 交互环境 | 快速验证语法、表达式和小片段 |

`python main.py` 和 `python -m package.module` 的区别：

- `python main.py`：按文件路径运行脚本，适合单文件或简单脚本
- `python -m package.module`：按模块路径运行入口，适合已经组织成包的项目

这里的 `-m` 是 `module` 的缩写，意思是：不要把后面的内容当作文件路径，而是当作 **Python** 模块路径

例如项目结构如下

```text
project/
  app/
    __init__.py
    main.py
    config.py
```

 `main.py` 里使用显式==相对导入==

```python
from .config import settings
```

这个 `.` 表示“从当前包里找 `config`”，它依赖一个前提：当前模块知道自己属于 `app` 包，通过内置变量 `__package__` 获取

- 直接执行文件路径时 `python app/main.py`
  - 此时 `__package__ = None`
  - **Python** 更倾向于把 `main.py` 当成独立脚本，入口模块没有稳定的包身份，所以这种显式相对导入正常会失败
- 按模块路径执行时 `python -m app.main`
  - 此时`__package__ = "app"`
  - **Python** 会把入口模块放回 `app` 包里执行，所以 `from .config import settings` 能按包结构解析

> 需要注意：如果代码使用普通绝对导入，并且模块在搜索路径里，`python app/main.py` 也可能正常工作
>
> ```python
> from app.config import settings
> ```
>

#### 1.4、`__name__` 和 `__main__`

- `__name__`：**Python** 给每个模块自动设置的内置变量，用来表示“当前模块的名字”
- `__main__`：**一个特殊模块名**，表示“当前正在作为程序入口执行的模块”

当一个文件被 **Python** 执行时，它可能有两种身份

- 被直接执行：这个文件是程序入口
- 被其他文件 `import`：这个文件只是被复用的模块

**Python** 会根据当前文件的使用方式，给 `__name__` 设置不同的值

| 当前文件的使用方式 | `__name__` 的值 | 是否应该执行 `main()` |
| --- | --- | --- |
| `python main.py` | `"__main__"` | 是 |
| `python -m package.main` | `"__main__"` | 是 |
| `import main` | `"main"` | 否 |
| `import package.main` | `"package.main"` | 否 |

示例：

```Python
# Python
def main() -> None:
    print("agent started")


if __name__ == "__main__":
    main()
```

这段代码的执行过程是

1. 如果执行 `python main.py`，`__name__` 等于 `"__main__"`，所以会调用 `main()`
2. 如果另一个文件执行 `import main`，`__name__` 等于 `"main"`，所以不会自动调用 `main()`

> 真实项目中应优先把可复用模块和可执行入口分开；
>
> if __name__ == "__main__" 主要用于保护入口文件，避免它被 import 时执行启动逻辑

### 二、缩进、代码块与命名

#### 2.1、语法与用法

==**Python** 用缩进表达代码块，代码块开头通常以 `:` 结束==

- `if condition:` 后面缩进的是条件成立时执行的代码
- `def name():` 后面缩进的是函数体
- `class Name:` 后面缩进的是类体
- 函数和变量通常使用 `snake_case` （下划线_命名）
- 类名通常使用 `PascalCase` （驼峰命名）

#### 2.2、示例代码

```python
# Python
def should_call_tool(tool_name: str) -> bool:
    if tool_name == "read_file":
        return True

    return False
```

```js
// JavaScript
function shouldCallTool(toolName) {
  if (toolName === "read_file") {
    return true;
  }

  return false;
}
```

### 三、变量绑定与可变性

#### 3.1、语法与用法

**Python** 的变量赋值是名字绑定，不区分 `let` 和 `const`

- `name = value` 表示把名字绑定到对象
- `list`、`dict` 是可变对象
- `str`、`int`、`tuple` 通常按不可变对象理解
- 是否能改，主要看对象本身是否可变，而不是变量声明方式

#### 3.2、示例代码

```python
# Python
profile = "default"
profile = "work"

tools = ["read_file"]
tools.append("write_file")
```

```js
// JavaScript
let profile = "default";
profile = "work";

const tools = ["read_file"];
tools.push("write_file");
```

### 四、基础类型与空值

#### 4.1、语法与用法

**Python** 常见基础类型如下

- `str`：字符串
- `int`：整数
- `float`：浮点数
- `bool`：布尔值，只能写 `True` 和 `False`
- `None`：空值，常用于表示没有结果

判断 `None` 推荐使用 `is None`，不要用 `== None`

#### 4.2、示例代码

```python
# Python
session_id: str | None = None

if session_id is None:
    session_id = "new-session"
```

```js
// JavaScript
let sessionId = null;

if (sessionId === null) {
  sessionId = "new-session";
}
```

### 五、条件、循环与 range

#### 5.1、语法与用法

- `elif` 对应 **JavaScript** 的 `else if`
- `for item in items` 用于遍历可迭代对象
- `break` 结束循环，`continue` 跳过本轮
- `range(n)` 返回一个可以被遍历的 `range` 对象，表示从 `0` 到 `n - 1` 的整数序列
- `range(start, stop, step)` 可以指定起点、终点和步长

#### 5.2、range

`range(start, stop, step)` 的参数含义如下

- `start`：起始值，省略时默认为 `0`
- `stop`：结束边界，不包含该值
- `step`：步长，省略时默认为 `1`

`range(...)` 返回一个 `range` 对象

这个对象表示一段整数序列，常用于 `for` 循环

```python
numbers = range(3)

for number in numbers:
    print(number)

# 输出结果是
# 0
# 1
# 2
```

如果确实需要把它展开成列表，可以显式调用 `list(...)`

```python
numbers = list(range(3))
# numbers = [0, 1, 2]
```

#### 5.3、示例代码

```python
# Python
tools = ["read_file", "write_file", "run_command"]

for index in range(len(tools)):
    tool_name = tools[index]

    if tool_name == "run_command":
        continue

    print(tool_name)
```

```js
// JavaScript
const tools = ["read_file", "write_file", "run_command"];

for (let index = 0; index < tools.length; index += 1) {
  const toolName = tools[index];

  if (toolName === "run_command") {
    continue;
  }

  console.log(toolName);
}
```

### 六、推导式、生成器表达式与 match / case

#### 6.1、列表推导式

**「列表推导式」**用于从一个可迭代对象生成新列表

```python
tools = [{"name": "read_file"}, {"name": "write_file"}]

tool_names = [tool["name"] for tool in tools]
# ["read_file", "write_file"]
```

这行代码可以拆成三步理解

1. `for tool in tools`：从 `tools` 里逐个取出 `tool`
2. `tool["name"]`：每次取出当前 `tool` 的 `name`
3. `[...]`：把每次计算出来的结果收集成新列表

它等价于下面的普通循环

```python
tools = [{"name": "read_file"}, {"name": "write_file"}]

tool_names = []

for tool in tools:
    tool_names.append(tool["name"])

# ["read_file", "write_file"]
```

#### 6.2、带条件的列表推导式

列表推导式后面可以加 `if`，用于过滤不需要的元素

```python
tools = [
    {"name": "read_file", "enabled": True},
    {"name": "write_file", "enabled": False},
]

enabled_tool_names = [tool["name"] for tool in tools if tool["enabled"]]
# ["read_file"]
```

这行代码的含义是：只保留 `enabled` 为 `True` 的工具，再取出它们的 `name`

#### 6.3、字典推导式

**「字典推导式」**用于从一个可迭代对象生成新字典

```python
tools = [{"name": "read_file"}, {"name": "write_file"}]

tool_map = {tool["name"]: tool for tool in tools}
# {"read_file": {"name": "read_file"}, "write_file": {"name": "write_file"}}
```

这行代码可以拆成三步理解

1. `for tool in tools`：从 `tools` 里逐个取出 `tool`
2. `tool["name"]`：作为新字典的 **key**
3. `tool`：作为新字典的 **value**

它等价于下面的普通循环

```python
tools = [{"name": "read_file"}, {"name": "write_file"}]

tool_map = {}

for tool in tools:
    tool_map[tool["name"]] = tool

# {"read_file": {"name": "read_file"}, "write_file": {"name": "write_file"}}
```

#### 6.4、生成器表达式

生成器表达式和列表推导式长得很像，但返回值不同

```python
tools = [{"name": "read_file"}, {"name": "write_file"}]

tool_names = (tool["name"] for tool in tools)
# 生成器对象，不是列表
```

列表推导式会立刻生成完整列表

```python
tool_names = [tool["name"] for tool in tools]
# ["read_file", "write_file"]
```

生成器表达式不会立刻生成完整列表，而是在被遍历时按需产生数据

```python
tool_names = (tool["name"] for tool in tools)

for tool_name in tool_names:
    print(tool_name)

# read_file
# write_file
```

如果确实需要把生成器表达式的结果展开成列表，可以使用 `list(...)`

```python
tool_names = list(tool["name"] for tool in tools)
# ["read_file", "write_file"]
```

#### 6.5、match / case

`match / case` 是结构化分支语法，可以先简单理解为更强的 `switch`

```python
tool = {"name": "read_file", "type": "safe"}

match tool["type"]:
    case "safe":
        risk_level = "low"
    case "dangerous":
        risk_level = "high"
    case _:
        risk_level = "unknown"

# "low"
```

这里的 `case _` 表示兜底分支，类似 **JavaScript** `switch` 里的 `default`

对应的 **JavaScript** 写法如下

```js
const tool = { name: "read_file", type: "safe" };

let riskLevel;

switch (tool.type) {
  case "safe":
    riskLevel = "low";
    break;
  case "dangerous":
    riskLevel = "high";
    break;
  default:
    riskLevel = "unknown";
}
// "low"
```
